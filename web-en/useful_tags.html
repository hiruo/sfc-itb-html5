Commonly Used HTML Tags

In lecture #6, we have discussed how to use various HTML tags. In this section, we would go over some of the most commonly used tags, and show examples.

Lists

The "ul" (unordered list), "ol" (ordered list), and "li" (list item) tags are useful among making a list in HTML. Items listed on the "ordered list" begin with a number, and items listed on the "unordered list" begin with a bullet. Upon listing items, you must use the "li" tag for each item.

  <ul>
    <li>Apple</li>
    <li>Banana</li>
    <li>Grape</li>
  </ul>
  <ol>
    <li>Item Number One</li>
    <li>Item Number Two</li>
    <li>Item Number Three</li>
  </ol>

Definition Lists

In order to create a list with a definition for each listed item, the use of "dl" (definition list), "dt" (definition term), and "dd" (definition/description) tags may be a good option. In definition lists, the "definition term" indicates a single listed item, and the "difinition" indicates the explanation or the description of the item. 

  <dl>
    <dt>HTML</dt>
    <dd>HyperText Markup Language</dd>

    <dt>JFK</dt>
    <dd>John F. Kennedy</dd>

    <dt>USA</dt>
    <dd>United States of America</dd>
  </dl>

Tables

When creating tables in HTML, the "table" tag would be something useful. New usage methods of the "table" tag was introduced for HTML5. However, in order to maintain backward compatibility with older browsers, we recommend you use the following format of code when creating tables. This format would pass the W3C Markup Validatorwork for HTML5, and work with non-HTML5 browsers, as well.

CSS 
  /* Style Definitions for Table */
  table {
    border: solid 1px black;
  }

  th {
    border: solid 1px gray;
    background-color: #eeeeee;
  }

  td {
    border: solid 1px gray;
  }
HTML 
  <table>
    <tr>
      <th>Name</th>
      <th>Age</th>
      <th>Hobby</th>
    </tr>
    <tr>
      <td>Bob</td>
      <td>21</td>
      <td>Tennis</td>
    </tr>
    <tr>
      <td>Alice</td>
      <td>19</td>
      <td>Basketball</td>
    </tr>
  </table>
